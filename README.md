# Gtk-Sharp-Samples

<h1>What is GTK#?</h1>
<p>
  Itâ€™s a wrapper on the Gimp Toolkit, or GTK+ a cross platform GUI framework. Currently it works natively for any X server or the native Windows UI.
</p>
<p>
Although Gtk# is not limited to C#, any programming language in the .NET framework can use it, because it covers more than only the GTK API, it covers various other components of the Gnome Desktop.
</p>
<p>
  Gtk# is made up of the following components (assemblies), each one mapping the corresponding library:
</p>
<table>
  <tr>
    <td>gtk-sharp</td>
    <td>Bindings to the gtk+ 2.x toolkit which is a
cross-platform toolkit for creating graphical
user interfaces.</td>
  </tr>
  <tr>
    <td>glib-sharp</td>
    <td>Bindings to the glib 2.x toolkit which provides
a low-level non-GUI core library for gtk+</td>
  </tr>
  <tr>
    <td>pango-sharp</td>
    <td>Bindings to the Pango (http://www.pango.org)
international text layout and rendering library.</td>
  </tr>
  <tr>
    <td>atk-sharp</td>
    <td>Bindings to the atk Accessibility framework.</td>
  </tr>
  <tr>
    <td>Gdk-sharp</td>
    <td>low-level Drawing Kit using by gtk+.</td>
  </tr>
  <tr>
    <td>Glade-sharp</td>
    <td>Glade# allows you to load Glade interface files
in your program at runtime. This is the easiest
way to create Gtk# GUIs.</td>
  </tr>
  <tr>
    <td>Art-sharp</td>
    <td>The vector and rendering library.</td>
  </tr>
  <tr>
    <td>gtk-dotnet</td>
    <td>Integration of the Gtk# namespace with System.
Drawing.</td>
  </tr>
  <tr>
    <td>Gnome-sharp</td>
    <td>Gnome bindings.</td>
  </tr>
  <tr>
    <td>Gconf-sharp</td>
    <td>Bindings to the configuration storage system
on Gnome</td>
  </tr>
  <tr>
    <td>Gtkhtml-sharp</td>
    <td>Bindings for the lightweight HTML widget.</td>
  </tr>
</table>
